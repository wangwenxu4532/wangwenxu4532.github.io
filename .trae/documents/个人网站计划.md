# 个人网站搭建计划

## 项目概述
为秋招打造一个免费的在线个人简历/作品集网站，基于纯静态 HTML/CSS/JS，部署到 GitHub Pages，实现浏览器直接访问（如 `wangwenxu.github.io`）。

## 当前状态分析
- **工作目录**: `d:\AI\Trea\个人网站 git`
- **已有材料**: 简历 PDF（`黑龙江东方学院-王文旭-电子信息工程专业.pdf`），内容已提取
- **缺失**: 网站代码、Git 仓库、GitHub 远程仓库、GitHub Pages 配置

## 决策与假设
1. **技术栈**: 纯静态单页网站（HTML5 + CSS3 + Vanilla JS），零依赖、零后端，最轻量且最适合 GitHub Pages。
2. **风格**: 现代蓝白清爽风，兼顾技术感与应届生活力，阅读体验优先。
3. **结构**: 单页应用（Single Page），顶部导航锚点跳转，支持平滑滚动。
4. **域名**: 使用 GitHub Pages 免费域名（`xxx.github.io`），暂不涉及自定义域名购买。
5. **响应式**: 必须适配手机、平板、桌面三种尺寸，HR/面试官可能用手机查看。

## 需要用户补充的材料（执行前提供）
1. **个人头像/照片**（可选）：是否有清晰证件照或技术风格头像？没有则使用姓名首字母占位。
2. **GitHub 账号**：是否有？没有则需注册，用于部署网站和展示代码。
3. **其他链接**（可选）：个人博客、CSDN、LinkedIn、Gitee 等。
4. **求职意向标题**：例如 "嵌入式软件工程师"、"硬件开发工程师"，用于网站大标题。

## 实施步骤

### Step 1: 初始化项目与 Git 仓库
- 创建 `index.html`、`css/style.css`、`js/main.js`、`assets/` 目录结构。
- 初始化 Git 本地仓库，配置 `.gitignore`。

### Step 2: 编写 HTML 结构与内容
基于已提取的简历，在 `index.html` 中构建以下区块（从上到下）：
1. **导航栏 (Navbar)**：Logo/姓名、各区块锚点链接、移动端汉堡菜单按钮。
2. **Hero 区域**：姓名、求职意向标题、一句话个人简介、CTA 按钮（查看项目 / 下载简历）。
3. **关于我 / 教育背景**：学校、专业、时间、排名、奖学金、主修课程。
4. **实践经历**：四福科技（硬件运维助理）、中建八局（测控开发助理），时间线排版。
5. **项目经历**：
   - C语言宿舍信息管理系统
   - 基于 STC89C52 环境监测系统
   - 全国大学生智能车竞赛项目
6. **技能栈**：C语言、单片机、PCB设计、Keil5/Proteus/Qt/Linux/MATLAB 等，可用标签/进度条展示。
7. **荣誉证书**：列表形式展示奖项。
8. **联系方式 / Footer**：电话、邮箱、GitHub 链接、微信二维码（可选）。
9. **下载简历按钮**：直接链接到本地 PDF 文件。

### Step 3: 编写 CSS 样式
- 定义设计令牌：主色调（科技蓝 `#2563EB`）、背景色、文字色、字体栈。
- 实现响应式网格布局（Flexbox + CSS Grid）。
- 实现导航栏滚动变色、移动端菜单展开/收起。
- 各区块间距、卡片阴影、Hover 动效。
- 打印样式（可选）：支持直接打印网页为简历格式。

### Step 4: 编写 JavaScript 交互
- 平滑滚动（点击导航自动滚动到对应区块）。
- 移动端菜单切换逻辑。
- 滚动时导航栏高亮当前区块。
- 简单的淡入动画（Intersection Observer 实现元素进入视口时渐显）。

### Step 5: 本地验证
- 在浏览器打开 `index.html`，检查各区块内容、排版、链接。
- 使用浏览器 DevTools 模拟手机/平板视图，确认响应式无错乱。
- 点击"下载简历"测试 PDF 下载是否正常。

### Step 6: GitHub 远程仓库与部署
- 在 GitHub 新建仓库（建议命名为 `wangwenxu.github.io`，如果用户名可用）。
- 将本地代码推送到 GitHub。
- 开启 GitHub Pages（Settings > Pages > Source 选择 main 分支）。
- 等待 1-3 分钟，验证 `https://<username>.github.io` 能否正常访问。
